### 已知缺口 ⚠️（不影响 dry-run 测试，影响实盘安全）

**`executor_buy.execute_rebuy()` 没有串入 Steam 价格校验**——`verify_steam_before_rebuy` 配置项存在但没有被调用。这意味着补仓时只看 C5 价，不验证 Steam 端是否崩。在测试阶段影响不大（箱子价格稳定），正式实盘前必须补上。

## 二、配置步骤

### Step 1：设置环境变量

在 Windows 系统环境变量（或项目根的 `.env`）里设置：

```
# 必填
C5GAME_API_KEY=你的C5 OpenAPI密钥

# Steam 执行层（上架才需要）
STEAM_COOKIES=steamLoginSecure=xxx; sessionid=yyy; steamid=zzz
STEAM_ID64=你的64位SteamID

# Steam Guard 2FA 自动确认（可选但强烈建议）
STEAM_IDENTITY_SECRET=你的identity_secret_base64
STEAM_DEVICE_ID=android:xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx

# 价格API（至少一个）
CSQAQ_API_TOKEN=xxx
STEAMDT_API_KEY=xxx

# 通知（可选）
SERVERCHAN_SENDKEY=xxx
```

> **`STEAM_COOKIES` 怎么拿**：用浏览器登录 Steam，F12 → Application → Cookies → `steamcommunity.com`，复制 `steamLoginSecure`、`sessionid`两个值，拼成 `key=val; key2=val2` 格式。
>
> **`STEAM_IDENTITY_SECRET` 怎么拿**：从 Steam Desktop Authenticator（SDA）或 WinAuth 导出的 `.maFile` 里有 `identity_secret` 字段，就是这个值。

### Step 2：初始化数据库

```bash
python main.py init-db
python main.py import-catalog
```

### Step 3：配置策略参数（关键）

编辑 `data/strategy_config.json`，**箱子测试专用配置**如下：

```json
{
  "executionEnabled": false,
  "dryRun": true,
  "autoListEnabled": true,
  "autoRebuyEnabled": false,
  "minPrice": 0.5,
  "guadaoMaxListingRatio": 0.73,
  "transferMinRealRatio": 0.95,
  "priceTolerancePct": 1.5,
  "maxListPerCycle": 1,
  "maxBuyPerCycle": 0,
  "cycleIntervalMinutes": 15,
  "listingCheckIntervalMinutes": 5,
  "forceRefreshBeforeExecution": true,
  "verifySteamBeforeRebuy": true,
  "rebuySteamDropTolerancePct": 3.0,
  "listingWallMinCount": 10,
  "listingPriceOffset": 0.01,
  "dryRun": true
}
```

> `maxBuyPerCycle: 0` + `autoRebuyEnabled: false` = 完全禁止自动补仓，只允许上架。最大限度降低风险。

### Step 4：把箱子加入底仓

先确认你有哪些箱子：

```bash
python main.py search-item "case"
python main.py search-item "箱"
```

然后加入底仓（用真实的 market_hash_name）：

```bash
python main.py pool-add "Danger Zone Case" --quantity 3
python main.py pool-add "Fracture Case" --quantity 3
```

## 三、测试流程（从安全到真实）

### 阶段 0：验证基础连通性（不涉及资金）

```bash
# 1. 检查 Steam 登录态
python main.py steam auth-check

# 2. 查看当前 Steam 挂单
python main.py steam listings

# 3. 手动查一个箱子的实时 Steam 价格（直连）
python main.py steam price-check "Danger Zone Case"

# 4. 扫描策略（只看不动）
python main.py pool scan
```

第五步：同步库存，只保留革命武器箱
# 同步全部 C5 库存
python main.py pool sync

# 看同步了什么
python main.py pool status

# 找到革命武器箱的准确 market_hash_name
python main.py search-item --keyword "Revolution"

把输出里的准确名称记下来（比如 Revolution Case），然后清理底仓只留箱子：

sqlite3 data/assistant.db "DELETE FROM inventory_pool WHERE market_hash_name != 'Revolution Case';"
sqlite3 data/assistant.db "DELETE FROM inventory_assets WHERE market_hash_name != 'Revolution Case';"

确认只剩箱子：

python main.py pool status

第六步：模拟跑（dry-run，不下单）
python main.py executor start --once --dry-run

观察输出：

出现 [dry-run] 上架 Revolution Case asset=xxx price=x.xx → 流程正常
出现 底仓为空 或扫描结果为空 → 检查 Step 5 是否成功
价格看起来合理（接近 Steam 当前最低卖价） → 可以进入真实测试
第七步：真实跑，最多上架 3 件
python main.py executor start --once --enable --no-dry-run --max-list 3 --max-buy 0

立刻检查：

# 看 Steam 上是否出现挂单
python main.py steam listings

# 看底仓状态，应该有几件变成 listed
python main.py pool status

如果 Steam 挂单出现且价格合理 → 测试通过。

如果需要手动确认 2FA（没配 STEAM_IDENTITY_SECRET）：

python main.py steam confirm

第八步：观察卖出 + 手动补仓
等箱子卖出后（几分钟到几小时），再查状态：

python main.py pool status

状态变成 pending_rebuy = 卖出成功，等待补仓。这时候不要开自动补仓，手动去 C5 补回来，观察整个流程是否符合预期。

整体节奏
auth-check → dry-run → 真实上架3件 → Steam确认有挂单 → 等卖出 → 手动补仓

每一步确认没问题再走下一步，不要跳
