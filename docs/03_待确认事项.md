# CS2 理财助手待确认事项 v0.1

这份文件不是阻塞项，而是后续继续完善需求时需要你拍板的地方。

## 1. 提醒渠道

已确认：第一版使用 `企业微信 ServerChan` 作为提醒渠道。

## 2. 自动执行的安全边界

已确认：第一版采用“提醒 + 用户确认”的交易模式。

具体规则：

1. 系统自动扫描价格和条件。
2. 命中买入或卖出条件后发送提醒。
3. 购买和卖出都需要用户确认后再执行。
4. 第一版不做无人值守自动买入和自动卖出。

## 3. 仓位账本口径

已确认：

1. 成本由用户人工记录。
2. 系统以用户录入为准，不强制按加权平均或最后一笔自动推算。

后续可选增强：

1. 保留卖飞后的目标接回价。
2. 支持平台外成交和人工补仓记录。

## 4. 指数计算方式

已确认：自定义篮子指数采用“篮子内饰品价格直接相加”的方式。

定义说明：

1. 指数值 = 篮子内所有饰品当前价格之和。
2. 第一版不做平均价、金额加权或件数加权。
3. 如果同一篮子内需要记录数量，后续再扩展为“价格乘数量后求和”。

## 5. 挂刀比例定义

已确认：挂刀比例公式如下。

`挂刀比例 = C5饰品价格 ÷ (0.869 × Steam饰品最低在售价)`

定义说明：

1. 分子使用 `C5` 饰品价格。
2. 分母使用 `Steam` 饰品最低在售价乘以 `0.869`。
3. `0.869` 视为第一版固定折算系数。
4. 第一版先围绕这个基础公式做提醒和候选筛选。

## 6. 优先级建议

如果按“最能立刻帮你交易”的顺序，我建议：

1. 单品监控 + 多阈值提醒
2. 仓位账本
3. C5 快速购买
4. C5 在售改价
5. 挂刀比例提醒
6. 系列指数和自定义篮子
