# CS2 理财助手需求规格说明书 v1.0

## 1. 产品目标

v1.0 只专攻一个核心模块：**基于我的库存，辅助我挂刀倒余额，并辅助我找出做 T 收益率最高的饰品类型**。

帮助用户围绕 CS2 饰品做交易决策，当前阶段重点覆盖以下场景：

1. 基于所有已绑定 `Steam` 账号库存，筛选适合挂刀倒余额的候选饰品类型。
2. 基于同一批库存，计算做 T 收益率并输出前十。
3. 结合 `C5` 与 `Steam` 的最低售价，给出挂刀比例和收益率结果。
4. 明确每个候选饰品对应的 `steamId` 与 `nickname`，方便用户定位库存来源。
5. 其他 watch、basket、仓位、自动化能力暂时都围绕这个核心模块服务。

当前进一步收敛后的落地重点：

1. 挂刀相关功能，当前只保留“挂刀比例监控 + 提醒”。
2. 做 T 相关功能，重点做“全库存收益率前 N + 价格门槛 + ServerChan 提醒”。
3. 提醒相关功能要单独做模块，和价格采集、规则判断、通知发送解耦。

## 2. 核心使用场景

### 2.1 单个饰品监控

用户选择一个具体饰品后，系统持续跟踪其价格、在售数量、求购价、涨跌幅，并支持：

1. 低于某个价格时提醒买入。
2. 高于某个价格时提醒卖出。
3. 下跌达到某个百分比时提醒。
4. 上涨达到某个百分比时提醒。
5. 同一个饰品设置多个提醒条件。

### 2.2 系列或篮子监控

用户希望监控一整个武器箱、印花系列、刀系、热门系列，或者自定义一篮子饰品，把它们当成“指数”观察。

系统需要支持两类指数：

1. 平台已有指数或系列。
2. 用户自定义篮子指数。

### 2.3 持仓与空仓记录

用户不只是看行情，还要知道自己现在是：

1. 持仓中。
2. 已卖出，空仓观察中。
3. 卖飞后等待回接。
4. 纯观察，没有持仓。

系统需要本地记录每个标的的状态、成本、仓位、预期回接价、目标卖出价、已实现收益和未实现收益。

### 2.4 C5 交易辅助

用户主要用 `C5` 交易，因此系统需要优先支持：

1. 查询当前最低价和可买库存。
2. 找到符合条件的在售列表。
3. 触发一键成交或快速购买。
4. 查看当前在售并做批量改价。
5. 在条件不满足时下架或停止自动操作。

第一阶段交易执行规则：

1. 命中买入条件后，系统只提醒，不自动下单。
2. 命中卖出条件后，系统只提醒，不自动卖出。
3. 购买和卖出都需要用户确认。

### 2.5 基于库存的挂刀倒余额 / 做T收益率模块

这是 v1.0 的核心模块，也是当前全项目优先级最高、最直接影响收益的功能模块。

用户会用 `C5 + Steam` 做余额搬运，因此系统要围绕“我的库存”提供两条直接可用的能力：

1. 根据我的库存，辅助我挂刀倒余额。
2. 根据我的库存，辅助我找出做 T 收益率前十的饰品类型。

当前用户的实际使用方式已经明确为：固定按 `0.73` 折倒余额，在 `Steam` 买入，在 `C5` 卖出做 T。

第一阶段已确认的基础公式：

`挂刀比例 = C5饰品价格 ÷ (0.869 × Steam饰品最低在售价)`

`做T收益率 = 挂刀比例 × 0.869 - 0.73`

等价写法：

`做T收益率 = (C5饰品价格 ÷ Steam饰品最低在售价) - 0.73`

模块实现要求：

1. 汇总所有已绑定 `Steam` 账号的库存。
2. 同类型饰品按 `marketHashName` 去重归并，同类型多件只算一个“饰品类型”。
3. 每个饰品类型同时计算挂刀比例和做 T 收益率。
4. 做 T 收益率结果按从高到低排序，默认输出前十。
5. 支持附加过滤条件：`C5` 最低售价大于等于指定阈值。
6. 如果存在 `C5` 价格但缺少 `Steam` 价格的饰品，必须单独记录并提示用户排查。
7. v1.0 输出字段只保留最核心信息，不输出多余说明。

做 T Top10 统一输出字段：

1. 饰品名称。
2. `marketHashName`。
3. 对应 `steamId`。
4. 对应 `nickname`。
5. 做 T 收益率（百分比）。
6. 挂刀比例。
7. `C5` 最低售价。
8. `Steam` 最低售价。

## 3. 功能模块

### 3.1 标的库

系统需要维护统一的标的主表，至少包含：

1. 中文名。
2. `marketHashName`。
3. 平台 `itemId`。
4. 分类信息。
5. 所属系列。

### 3.2 库存辅助挂刀 / 做T 模块

这是 v1.0 的主模块，其他模块优先级都低于它。

模块拆成两条主能力：

1. 库存辅助挂刀倒余额。
   - 从所有已绑定 `Steam` 账号拉取库存。
   - 聚合同类型饰品。
   - 计算挂刀比例。
   - 找出更适合倒余额的库存候选品。
2. 库存辅助做 T 收益率 Top10。
   - 对库存中的每个饰品类型计算做 T 收益率。
   - 按收益率排序。
   - 输出前十并标注所属账号。

### 3.3 价格采集

系统需要定时采集以下维度：

1. C5 最低在售价。
2. C5 在售数量。
3. C5 求购最高价。
4. Steam 价格或 Steam 近 7 天均价。
5. `CSQAQ` 的 `Steam` 价格补充、系列、指数、排行和挂刀数据。
6. 用户所有已绑定 `Steam` 账号的库存。
7. 按 `marketHashName` 聚合后的库存饰品类型池。

价格源优先级说明：

1. 库存做 T 模块中的 `C5` 价格优先使用库存查询结果自带价格。
2. `Steam` 价格优先用批量接口获取。
3. 如果存在 `C5` 价格但 `Steam` 价格缺失，系统必须记录问题并提示。

### 3.4 提醒引擎

系统支持四类提醒规则：

1. 绝对价格提醒。
2. 涨跌幅提醒。
3. 比例提醒。
4. 仓位状态联动提醒。

此外，提醒引擎还需要补充一条独立于通用规则引擎的“做 T 汇总提醒”能力：

1. 检查 `C5` 全部绑定账号全部库存。
2. 计算做 T 收益率。
3. 按 `Top N + C5价格 > xx` 过滤。
4. 每天早、中、晚各发送一次 `ServerChan` 汇总提醒。

提醒规则要支持：

1. 一个标的配置多条规则。
2. 买入提醒与卖出提醒分开配置。
3. 冷却时间，避免反复轰炸。
4. 一次触发后保留历史记录。

提醒模块拆分要求：

1. 价格采集与计算放在市场/策略模块。
2. 规则判定放在提醒规则模块。
3. `ServerChan` 等通知渠道放在通知模块。
4. 做 T 汇总提醒单独做服务层，后续便于加更多提醒类型。

### 3.5 仓位账本

API 无法完整表达你的真实交易成本和“卖飞后计划接回”的主观策略，因此系统必须有本地仓位账本。

成本口径说明：

1. 第一阶段由用户人工记录成本。
2. 系统以手工录入结果为准，不自动推导真实成本。

建议账本字段：

1. 标的。
2. 状态。
3. 持仓数量。
4. 持仓成本。
5. 计划买回价。
6. 计划卖出价。
7. 已实现收益。
8. 未实现收益。
9. 备注。

### 3.6 交易执行层

执行层以 `C5` 为主，优先支持：

1. 快速购买。
2. 条件筛选后购买。
3. 查看在售。
4. 改价。
5. 下架。

第一阶段建议采用“提醒 + 人工确认 + 一键执行”的模式，不建议一上来做纯无人值守自动下单。

### 3.7 通知中心

后续应支持多个通知渠道：

1. 桌面提醒。
2. Telegram。
3. 企业微信或微信转发。
4. 邮件。

第一阶段已确认使用 `企业微信 ServerChan`。

## 4. 业务规则建议

### 4.1 指数规则

指数分成两种：

1. 官方指数或热门系列指数，直接使用平台提供的数据。
2. 自定义指数，按用户手选饰品做平均价、加权价或总市值指数。

第一阶段自定义指数口径已确认：

1. 指数值 = 一篮子饰品当前价格直接相加。
2. 第一阶段不做平均价和权重计算。

### 4.2 提醒判定规则

建议提醒条件支持这些写法：

1. `价格 <= 100` 时提醒买入。
2. `价格 >= 130` 时提醒卖出。
3. `24h 跌幅 <= -6%` 时提醒。
4. `从最近高点回撤 >= 10%` 时提醒。
5. `挂刀比例 <= 0.88` 或 `>= 0.96` 时提醒。
6. `做T收益率 >= 6%` 时提醒优先关注。

### 4.3 仓位状态

建议用以下状态：

1. `holding`
2. `watch_only`
3. `waiting_reentry`
4. `sold_out`
5. `disabled`

### 4.4 风控规则

自动执行前至少要检查：

1. C5 余额是否足够。
2. 目标价格是否仍然满足条件。
3. 该提醒是否处于冷却中。
4. 是否超过单笔和单日预算。
5. 是否需要自动发货条件。

补充说明：

1. 第一阶段所有买入和卖出动作都需要用户确认。
2. 第一阶段不启用无人值守自动交易。

## 5. 分阶段落地建议

### 第一阶段（v1.0）

v1.0 专攻“库存辅助挂刀 / 做T 模块”，其他功能只保留必要配套：

1. 所有已绑定 `Steam` 账号库存汇总。
2. 同类型饰品聚合。
3. 基于库存的挂刀倒余额辅助。
4. 基于库存的做 T 收益率 Top10 排名。
5. 基于 `Top N + C5价格门槛` 的做 T 汇总提醒。
6. 缺少 `Steam` 价格的异常记录与查看命令。
7. 必要的价格采集、提醒和一键执行入口。

### 第二阶段

扩展到组合和策略层：

1. 系列指数监控。
2. 自定义篮子指数。
3. 更完整的挂刀候选池与分层排序。
4. 基于库存与趋势的联动筛选。

### 第三阶段

再做自动化与策略强化：

1. 自动扫货规则。
2. 自动挂售改价。
3. 订单回放和盈亏分析。
4. 更完整的风控系统。

## 6. 当前结论

这个产品当前最适合定义成“以库存辅助挂刀倒余额 / 做 T 为核心赚钱模块，以 `C5` 为交易执行核心，以 `SteamDT + CSQAQ` 为价格补充”的工具，而不是一开始就做成大而全的行情站或量化机器人。

v1.0 最重要的不是把所有接口都接完，而是先跑通这条链路：

`全账号库存汇总 -> 同类型聚合 -> 价格采集 -> 挂刀比例 / 做T收益率计算 -> Top10 输出 -> C5 手动执行`
